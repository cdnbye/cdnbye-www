    SwarmCloud

    基於ServiceWorker的HLS加速引擎，適配所有瀏覽器（包括iOS Safari）和Web播放器！

    iOS的Safari瀏覽器雖然可以支持原生播放HLS，但整個過程對開發者是透明的。Safari從2017年起就支持 WebRTC，但到目前為止還不支持 MEDIA SOURCE EXTENSIONS，而這個API對實現客戶端P2P至關重要。
    幸運的是，我們可以通過另一種方式來實現對HTTP流量的本地代理，從而間接實現P2P功能，這個用於實現本地代理的技術就是"ServiceWorkers"，本項目基於ServiceWorkers和WebRTC在所有現代瀏覽器和Web播放器實現了HLS的P2P能力。

    # 瀏覽器支持情況

    由於WebRTC已成為HTML5標準，目前大部分主流瀏覽器都已經支持。兼容性取決於瀏覽器是否支持 WebRTC 和 ServiceWorker。

    兼容性 Chrome Firefox macOS Safari Opera Edge iOS Safari IE
    WebRTC Datachannel ✔ ✔ ✔ ✔ ✔ ✔ ❌
    Service Worker ✔ ✔ ✔ ✔ ✔ ✔ ❌
    SwarmCloud ✔ ✔ ✔ ✔ ✔ ✔ ❌

    在安卓端，由于video标签直接播放HLS无法拦截媒体文件，因此需要用基于 MSE 的播放器如 hls.js 才能使本SDK正常工作

    # 工作原理

    • 首先由客戶端嵌入的SDK 註冊 ServiceWorker ，從而在播放器與網絡之間嵌入一層本地代理
    • 播放器向本地代理請求視頻流
    • 本地代理（ServiceWorker）通過監聽 fetch 事件，攔截全站的網絡請求，並識別出m3u8和ts文件請求，將請求轉發到主線程
    self.addEventListener('fetch', event => { /* ⛦ magic here ⛦ */ })
    • 主線程根據P2P網絡情況決定是否從對等端下載，下載完成後在主線程保留一份拷貝以分享給其他節點
    • 當資源被一個設備緩存之後，其他設備在請求相同資源時，將嘗試優先通過P2P網絡就近從該設備獲取資源，而不是從成本更高、距離更遠的源服務器獲取（即用WebRTC替代HTTP）
    • 從P2P網絡獲取失敗或者不能緩存的資源，將自動回源

    # 局限性

    • 由於ServiceWorker的限製，需要用https訪問網站
    • 由於ServiceWorker的限製，需要將sw.js文件部署至托管域名的服務器
    • 暂不支持 LL-HLS
