    SwarmCloud

    本SDK目前仅维护不再新增功能，且不支持iOS浏览器，推荐使用新一代SDK (opens new window)

    基於MSE和WebRTC技術的視頻網站省流量&加速引擎，適用於所有以Hls.js作為解碼引擎的播放器。

    # 目前最好的Web端P2P流媒體方案

    P2P技術使觀看相同內容的用戶之間可以相互分享數據，不僅能效降低視頻/直播網站的帶寬成本，還可以提升用戶的播放體驗，降低卡頓、二次緩存的發生率。
    另外，隨著H5的普及，flash逐漸被淘汰已成為不可逆轉的趨勢。而在H5采用的視頻傳輸格式中，hls由於兼容iOS和Android、可以穿過任何允許HTTP數據通過的防火墻、容易使用內容分發網絡來傳輸媒體流和碼率自適應等眾多優勢而在業界得到廣泛使用。通過使用hls.js (opens new window)這個第三方庫，幾乎所有現代瀏覽器都可以播放hls視頻。hls天生分片傳輸的優勢，使其可以采用p2p的方式進行傳輸，從而減小服務器的負擔。
    在Web端，無插件化實現p2p傳輸能力的最好選擇就是WebRTC (opens new window)技術，與hls.js類似，WebRTC也支持幾乎所有現代瀏覽器。本項目是一個hls.js的插件，通過WebRTC datachannel技術，在不影響用戶體驗的前提下，最大化p2p率，是面向未來的Web P2P技術。

    # 特性

    • 瀏覽器原生支持，不需要安裝任何插件，采用仿BT算法，在線人數越多效果越好
    • 支持基於HLS流媒體協議(m3u8)的直播和點播場景（如何將MP4轉成HLS? (opens new window)）
    • 支持加密HLS傳輸和防盜鏈技術
    • 不改動hls.js源碼，並且可以與其無縫銜接，幾行代碼集成，便於在現有項目中快速集成
    • 瀏覽器不支持WebRTC時無縫切換到HTTP下載模式
    • 配置靈活，用戶可以根據特定的使用環境調整各個參數
    • 支持video.js、Clappr、Flowplayer等第三方播放器
    • 通過有效的調度策略來保證用戶的播放體驗以及p2p率
    • Tracker服務器根據訪問IP的ISP、地域等進行智能調度

    # 瀏覽器支持情況

    由於WebRTC已成為HTML5標準，目前大部分主流瀏覽器都已經支持。CDNBye的瀏覽器兼容性取決於WebRTC和hls.js。需要註意的是iOS版Safari由於不支持MediaSource API，因此也不支持hls.js(不過Safari原生支持HLS播放)。

    兼容性 Chrome Firefox Mac Safari 安卓微信/QQ Opera Edge IE iOS Safari
    WebRTC ✔ ✔ ✔ ✔ ✔ ✔ ❌ ✔
    Hls.js ✔ ✔ ✔ ✔ ✔ ✔ ✔ ❌
    CDNBye ✔ ✔ ✔ ✔ ✔ ✔ ❌ ❌

    # 快速集成步驟

    • 在管理面板 (opens new window)點擊 網站管理，按照提示綁定域名
    • 選擇一個網站目前在用的播放器（例如DPlayer）
    • 將播放器部分代碼替換成demo的代碼，修改播放地址。
    • p2p插件也可以本地化 (opens new window)，或者用 npm 的方式安裝依賴
    • 調試通過後部署到服務器
    • 如果P2P不起作用，請參照問題排查步驟進行排查

    # 局限性

    • 在音頻和視頻軌分離的情況下，只有視頻軌能獲得P2P加速
    • 不能同時為2個以上(含2個)視頻進行P2P加速
    • 暂不支持 LL-HLS
