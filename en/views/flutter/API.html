# P2P Configuration

Instantiate P2pConfig ：

/// The configuration of p2p engine.
class P2pConfig {
  /// Print log level
  final P2pLogLevel logLevel;
  /// Configuration for WebRTC
  final Map<String, dynamic> webRTCConfig;
  /// The address of tracker server
  final String? announce;
  /// The max size of binary data that can be stored in the disk cache for VOD(Set to 0 will disable disk cache)
  final int diskCacheLimit;
  /// The max count of ts files that can be stored in the memory cache, only for Android
  final int memoryCacheCountLimit;
  /// @Deprecated('Use memoryCacheCountLimit now')
  /// iOS Only
  /// The max size of binary data that can be stored in the memory cache, only for iOS
  final int memoryCacheLimit;
  /// Enable or disable p2p engine
  final bool p2pEnabled;
  /// TS file download timeout by HTTP
  final Duration downloadTimeout;
  /// Add a custom label to every different user session, which in turn will provide you the ability to have more meaningful analysis of the data gathered
  final String? tag;
  /// The port for local http server(default is random port)
  final int localPort;
  /// Max peer connections at the same time
  final int maxPeerConnections;
  /// Use HTTP ranges requests where it is possible. Allows to continue (and not start over) aborted P2P downloads over HTTP
  final bool useHttpRange;
  /// Set HTTP Headers while requesting ts and m3u8
  final Map<String, String>? httpHeaders;
  /// Set it as true if SDK is running on set-top box
  final bool isSetTopBox;
  /// Time for HTTP download if p2p download timeout
  final int httpLoadTime;
  /// Allow the P2P transmission of m3u8 file
  final bool sharePlaylist;
  /// Save logs to the file(Please check the corresponding native platform document for the save location)
  final bool logPersistent;
  /// iOS Only
  /// The path to the log files, default is Library/Caches/Logs/
  final String? logFilePathInIos;
  /// Android Only
  final bool waitForPeerInAndroid;
  /// Android Only
  /// Try downloading initial segments over P2P, giving the peers extra time to download initial segments will increase the p2p Ratio
  final int waitForPeerTimeoutInAndroid;
  /// All media suffixes that allow P2P transmission
  final List<String> hlsMediaFiles;
  /// The country enum for the tracker server address(China, HongKong, USA) 
  final AnnounceLocation? announceLocation;
  /// Any media extension that allow P2P transmission
  final List<String>? hlsMediaFileExtensions;
  P2pConfig({
    this.logLevel: P2pLogLevel.warn,
    this.webRTCConfig: const {},
    this.wsSignalerAddr, //: 'wss://signal.cdnbye.com',
    this.announce, //: 'https://tracker.cdnbye.com/v1',
    this.diskCacheLimit: 1024 * 1024 * 1024,
    this.memoryCacheLimit: 100 * 1024 * 1024,
    this.memoryCacheCountLimit: 30,
    this.p2pEnabled: true,
    this.downloadTimeout: const Duration(seconds: 30),
    this.dcDownloadTimeout: const Duration(seconds: 6),
    this.tag,
    this.localPort: 0,
    this.maxPeerConnections: 20,
    this.useHttpRange: true,
    this.wifiOnly: false,
    this.httpHeaders,
    this.isSetTopBox: false,
    this.httpLoadTime: 2000,
    this.logPersistent: false,
    this.sharePlaylist: false,
    this.waitForPeerInAndroid: false,
    this.waitForPeerTimeoutInAndroid: 4500,
    this.hlsMediaFiles: const ["ts", "mp4", "m4s"],
    this.logFilePathInIos,
    this.announceLocation,
    this.hlsMediaFileExtensions,
  });
  P2pConfig.byDefault() : this();
}

# P2pEngine

Instantiate P2pEngine，which is a singleton:

SwarmCloud.init(
  token, /// replace with your token
  config: P2pConfig.byDefault()
);

Explanation:

param type required description
token String Yes Token assigned by SwarmCloud
config P2pConfig No Custom configuration

# Switch Stream URL

When switching to a new stream URL, before passing new stream url(m3u8) to the player, pass that URL through SwarmCloud :

String parsedUrl = await SwarmCloud.parseStreamURL(url);

# Callback Player Stats

On HLS Live streaming, to improve performance, we recommend to tell p2p engine the duration from the playback time to the end of the buffered interval. In order to do so, you need to use callback bufferedDurationGenerator .

SwarmCloud.init(
  token, /// replace with your token
  bufferedDurationGeneratorEnable: true,
);
String parsedUrl = await SwarmCloud.parseStreamURL(
  url,
  bufferedDurationGenerator: () {
    return vpController!.value.buffered.last.end - vpController!.value.position;
  },
);

# Dynamic m3u8 Path Support

The channelId is an identifier used by our backend to match peers that are watching the same content. It is an optional parameter, and by default, we generate channelId from the content URL by removing any query parameters and protocol from it. Some m3u8 urls play the same live/vod but have different paths on them. For example, example.com/clientId1/streamId.m3u8 and example.com/clientId2/streamId.m3u8. In this case, you can format a common channelId for them.

SwarmCloud.init(
  token,   /// replace with your token
  segmentIdGenerator: (streamId, int sn, String segmentUrl, String? range) {
    return '$sn';
  }
);
// 根据url构造videoId
String videoId = extractVideoIdFromUrl(originalUrl);     /// extractVideoIdFromUrl 需要自己定义，可以抽取url中的视频ID作为结果返回
String url = await SwarmCloud.parseStreamURL(originalUrl, videoId);

Interconnect with other platform should ensure that both have the same token and channelId.

# Dynamic Ts Path Support

Like dynamic m3u8 path, you should format a common segmentId for the same ts file. You can override the segment ID like this:

SwarmCloud.init(
  token,   /// replace with your token
  segmentIdGenerator: (streamId, int sn, String segmentUrl, String? range) {
    return '$sn';
  }
);

# Setup HTTP headers

Some HTTP requests need to add header information such as referer or User-Agent for Anti-Leech or statistical requirements. It can be set via httpHeaders :

P2pConfig(
  httpHeaders: {
    "referer": "XXX",
    "User-Agent": "XXX",
  }
)
