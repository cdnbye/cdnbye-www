# Prerequisites

# CORS

Make sure your streaming servers have proper CORS (Cross-origin resource sharing) headers so that data can be fetched across domain.

# OPTIONS REQUEST

OPTIONS requests are mandatory to be able to perform RANGE requests in a cross-domain environment. The general idea is to add the following header to the HTTP response:

Access-Control-Allow-Methods: GET, OPTIONS

# RANGE REQUEST

RANGE requests is required by P2P. Add the following header to the HTTP response:

Access-Control-Allow-Headers: Range

# Register your Domain

Register your domain at https://oms.cdnbye.com

Localhost is always whitelisted. This means that you do not have to configure anything to perform tests locally.

# Secure your Site with HTTPS

Secure your site with HTTPS, if it isn't already. HTTPS is required for Service Worker, which we'll set up in the next step.

To secure your site with HTTPS, you can use Let's Encrypt (opens new window) for a free certificate and easy integration. See Let's Encrypt's starting instructions here (opens new window) to secure your site.

# Host Service Worker

To maintain a high-security standard, foreign software to be integrated into the browser with the help of ServiceWorkers is not allowed. This is only possible if the software is hosted on the same server as the web page. So the ServiceWorkers only work if they are hosted under the same domain as the main page, meaning that a client running a website needs to host our web SDK on his own servers instead of getting it from our public cloud.

# If you don't have a Service Worker

Copy sw.js (opens new window) to your server and make it available at https://your_website.com/sw.js .

# If you already have a Service Worker

Integrate mp4-proxy.js into your existing Service Worker by adding this code to the beginning of your Service Worker's root JavaScript file:

self.importScripts('https://cdn.jsdelivr.net/npm/swarmcloud-mp4-sw@latest/dist/mp4-proxy.js')

Once added, 1) your existing Service Worker will continue to be installed, and 2) SwarmCloud's imported Service Worker script will handle CDN 'fetch' events to intercept MP4/MP3 request. All 'fetch' events not handled by SwarmCloud's Service Worker will fall through to your existing Service Worker's 'fetch' event handler(s).

# Install P2P Engine

Add P2P Engine script to the <head> element of home page. Again, you can customize configuration or use default one.

# Basic Usage

Add one line of script to your website homepage index.html, this will setup p2p engine automatically with default configuration.

<head>
    <script src="https://cdn.jsdelivr.net/npm/swarmcloud-mp4-sw@latest"></script>
    ...
</head>

# Advanced Usage

Add the following line as early as possible to the <head> tag of your index.html, you can pass any of the Client Config options to the P2PEngineMP4 class constructor:

<script src="https://cdn.jsdelivr.net/npm/swarmcloud-mp4-sw@latest"></script>

Or via npm

$ npm install swarmcloud-mp4-sw

import P2PEngineMP4 from 'swarmcloud-mp4-sw';
// Create P2PEngineMP4 instance...
window.disableP2PEngineMP4AutoInit = true;
var engine = new P2PEngineMP4({
    // logLevel: 'debug',
    // swFile: './sw.js',     // you may need to change it to the actual path of sw.js
    // Other p2pConfig options provided by P2PEngineMP4
})
engine.registerServiceWorker().then(function (registration) {
    console.info('ServiceWorker registration successful with scope: ', registration.scope);
}).catch((err) => {
    console.info('ServiceWorker registration failed ', err)
})

# Check if SwarmCloud works correctly

Open Chrome Console, if it prints like "ServiceWorker registration successful with scope", that means SwarmCloud is already working.

# Best Practices

  • Prepare
    • SSL enabled on your website
    • A CDNBye Portal Account
    • Your domain registered on the CDNBye Portal
  • Redirect user's HTTP request to HTTPS, so as to ensure that P2P can work normally
  • Put the sw.js (opens new window) file on the directory where the player script is
  • Then put P2PEngineMP4 SDK(ios-p2p-engine.min.js) on the playing page
<script src="https://cdn.jsdelivr.net/npm/swarmcloud-mp4-sw@latest"></script>

It will setup p2p engine automatically with default configuration, you can instantiate P2PEngineMP4 by yourself to customize p2p config:

window.disableP2pEngineMP4AutoInit = true;
var engine = new P2PEngineMP4({
    swFile: './sw.js',          // please set the correct path of sw.js
    // logLevel: 'debug',
    // Other p2pConfig options provided by P2PEngineMP4
})
engine.registerServiceWorker().then(function (registration) {
    console.info('ServiceWorker registration successful with scope: ', registration.scope);
}).catch((err) => {
    console.info('ServiceWorker registration failed ', err)
})
